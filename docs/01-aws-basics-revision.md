# AWS Basics Revision Notes

## Key Concepts

### 1. **AWS Overview**
   - AWS (Amazon Web Services) is a comprehensive cloud computing platform provided by Amazon.
   - It offers a wide range of services including computing power, storage options, and networking capabilities.

### 2. **Core Services**
   - **EC2 (Elastic Compute Cloud)**: Virtual servers in the cloud for running applications.
   - **S3 (Simple Storage Service)**: Scalable storage for data backup, archiving, and analytics.
   - **RDS (Relational Database Service)**: Managed relational database service for SQL databases.
   - **VPC (Virtual Private Cloud)**: Isolated network environment to launch AWS resources.
   - **IAM (Identity and Access Management)**: Manage access to AWS services and resources securely.

### 3. **Networking Basics**
   - **Subnets**: Segments of a VPC that can host resources.
   - **Security Groups**: Virtual firewalls that control inbound and outbound traffic to AWS resources.
   - **Route Tables**: Control the routing of traffic within a VPC.

### 4. **Storage Options**
   - **S3 Buckets**: Used for storing objects and files.
   - **EBS (Elastic Block Store)**: Block storage for use with EC2 instances.
   - **Glacier**: Low-cost cloud storage service for data archiving and long-term backup.

### 5. **Monitoring and Management**
   - **CloudWatch**: Monitoring service for AWS cloud resources and applications.
   - **CloudTrail**: Tracks user activity and API usage across AWS infrastructure.

### 6. **Security Best Practices**
   - Use IAM roles and policies to manage permissions.
   - Enable multi-factor authentication (MFA) for AWS accounts.
   - Regularly review and rotate access keys.

### 7. **Cost Management**
   - Understand the AWS pricing model and use the AWS Pricing Calculator.
   - Monitor usage and set up billing alerts to avoid unexpected charges.
